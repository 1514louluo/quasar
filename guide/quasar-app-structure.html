Overview

This is what a new Quasar Framework App folder structure looks like. Some folders are creating after building the App.

What each is used for:

AssetDescription
/buildDevelopment build folder
/distProduction build folder
/srcApp source files; see Source Folder
/testTest source files
/quasar.build.ymlYAML file used to configure App build

quasar.build.yml

This is the place to include your own dependencies, or configure the banner that is automatically added when building for Production, and many more.

Note that the format must be YAML.

Source Folder

AssetDescription
/src/cssFolder to store global/common CSS files
/src/htmlFolder to store global/common HTML files
/src/jsFolder to store global/common JS files
/src/pagesFolder to store assets for each page
/src/index.htmlApp starting point

Pages

An App’s central working point is the Pages it is composed of. Each page has the following structure:

AssetDescription
/src/pages/<page-name>Page folder
/src/pages/<page-name>/assetsFolder to place images, fonts, … specific to the page only
/src/pages/<page-name>/cssFolder to place CSS files specific to the page only
/src/pages/<page-name>/htmlFolder to place HTML files specific to the page only
/src/pages/<page-name>/jsFolder to place JS files specific to the page only
/src/pages/<page-name>/config.<page-name>.ymlFolder to place CSS files specific to the page only

Each page has a starting point for each type of asset:

AssetDescription
style.<page-name>.stylIf it exists, it will automatically be included when page is loaded.
When user navigates away, it is removed.
view.<page-name>.htmlIf it exists, it will automatically be included when page is loaded.
script.<page-name>.js[Required] Starting point of the page logic
config.<page-name>.yml[Required] YAML file with page configuration (called Manifest).

config.page-name.yml

Provide page specific configuration, called Manifest.

Important properties below. These get computed by default if CSS and HTML main page files exists, but they can be overriden to point to other files. The path must start with the folder where root index.html file exists.

Any other property here will get into the page manifest. Read more in the Writing a Page section.
